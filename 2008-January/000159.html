Hi all,

To help everyone not using emacs (hint :-) ), I have added file
templates in the '_templates' folder. This also has an 'add.sh' script
which is really simple to use.

Please, whenever adding a file to the repository, use 'add.sh'. This
will 

 a) Ensure the file is formatted correctly
 b) Add the file to the repository using 'svn add'
 c) Set the 'svn:keywords' property

To automatically set the @AUTHER@ field, just set the environment
variable 'SENF_AUTHOR' in your .bashrc or wherever you want.

To add a file to the current directory all you need to do is:

 $ ../../_templates/add.sh MyNewClass.hh

And if you place the following code into you'r .bashrc:

 senf_add () {
     local addsh
     addsh="_templates/add.sh"
     while [ ! -r "$addsh" ]; do adsh="../$addsh"; done
     $addsh "$@"
 }

This is simplified to

 $ senf_add MyNewClass.hh

(ok, this code will loop forever if called from without the SENF
repository but I don't think, thats a problem).

And if you inadvertently added the file, do

 $ senf revert MyNewClass.hh

to remove it from the repository again (that is, before it is
commited).

So to SUMMARIZE:

 - Everyone edit their .bashrc and add SENF_AUTHOR + above senf_add()
   function
 - Everyone only use 'senf_add' to add files to the repository

Hope this helps :-)

stefan.
