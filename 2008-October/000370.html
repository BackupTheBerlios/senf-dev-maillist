Hi,

I have just tested linux multicast performance using the 'multimcloop'
test tool from 'Examples/MultiMCLoop'.

Running one instance in the foreground and 70 instances with
redirected output in the background will produce a 50% load (1 core)
on my Core2 duo system.

CPU usage is mainly system, the user cpu load is below 4%.

So for thousands of hosts in our CARMEN simulation, we may need to
optimize but to get rolling, the one-socket-per-interface setup will
be ok.
