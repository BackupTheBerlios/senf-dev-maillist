I have started to implement the SSM API in the Socket
library. However, somehow every operation to leave an SSM multicast
group fails with EADDRNOTAVAIL:

    senf::UDPv6ClientSocketHandle sock;

    sock.protocol().mcJoinSSMSource(
                             senf::INet6Address::from_string("ff1e::1"),
                             senf::INet6Address::from_string("2001:db8::1"),
                             "eth") );

    // This fails with EADDRNOTAVAIL ...
    sock.protocol().mcLeaveSSMSource(
                             senf::INet6Address::from_string("ff1e::1"),
                             senf::INet6Address::from_string("2001:db8::1"),
                             "eth") );

This needs to be analyzed further.
