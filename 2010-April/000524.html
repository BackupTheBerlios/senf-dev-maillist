All,

I've changed the default (!debug and !final) compiler optimization options
to -O2 (from -O0). This should be in line with recommendations by the boost 
people and makes a standard senf binary run much more smoothly while still 
having run-time checks enabled.

Hence, use:

debug=1 to really debug a problem
final=1 to build fully optimized code
(nothing) for regular development builds

Sorry, for this information 'after the fact'.

Comment, flames, etc. welcome !
