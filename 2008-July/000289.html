I have just completed the implementation of the scheduler watchdog and
at the moment, I am using another signal, SIGURG for this.

This signal is normally used to notify the application of out-of-band
date on a socket. However, in the scheduler context using signals for
this notification does not make sense. Therefore I should be able to
reuse this signal.

I could possibly make it use SIGALRM too but that would be pretty
awkward. Though I don't like signals, in a single-threaded
application, the watchdog needs to be based on one.
