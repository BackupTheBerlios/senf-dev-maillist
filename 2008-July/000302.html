After discussing with Thorsten, I have opened a release branch for the
time being just named '0.1' so I can commit the API changes.

We can easily open a specific daidalos branch or some other named
branch from this branch. However, I prefer just to use numbered
releases:

 - When building debian packages, they will be correctly
   numbered. This is not possible with non-numerically named releases
   
 - The branch is not really daidalos specific, it's just an API
   freeze.

However, still it would be nice for to have a central place for
daidalos to specify the version in use.

As far as I understand, all daidalos code is also managed via svn. I
would propose to have a project or directory which is somehow called
'SENF' in that same repository. That project just has a single
svn:external entry which pulls in the correct SENF branch. The other
projects will then reference this SENF directory. To change the SENF
Version daidalos is using, just edit the svn:external property to
reflect that change.

For now, to directly use that Version, go to the top of your senf
checkout and do:

 $ svn switch https://svn.berlios.de/svnroot/repos/senf/branches/0.1

That's it. To switch back to the trunk, run

 $ svn switch https://svn.berlios.de/svnroot/repos/senf/trunk

I would however think, that integrating the API changes should be very
simple. You will also benefit from correctly formatted log message
times and from much higher resolution timers.

stefan.
