Stefan Bund schrieb:
> Thorsten Horstmann <thorsten.horstmann at fokus.fraunhofer.de> writes:
>> My Proposal:
>> we exclude everything in the senf::*::detail namespaces from the default 
>> documentation (i.e. senf.berlios.de). The class index pages will be 
>> cleaned up and the "normal" senf-user / -developer hopefully doesn't 
>> need the documentation from the classes in this namespaces.
> 
> Very good idea. Sadly, this is not simple since we then need to
> faithfully surround all code from the detail namespaces with #ifndef
> DOXYGEN / #endif preprocessor statements (not only in the header files
> but also in the implementation files)

nope, there is an undocumented option in doxygen: EXCLUDE_SYMBOLS
see the changelog for release 1.5.2:
"Added a new config option EXCLUDE_SYMBOLS which can be used to exclude 
certain namespaces/classes/functions from the output by specifying their 
name (which may include wildcards)."
