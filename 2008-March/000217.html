I have read up on readline (and some other libraries providing
readline like functionality). No library can easily be integrated. The
problems are:

- readline manages all it's state in global variables. However, the
  number of concurrent console clients is not limited, therefore it is
  absolutely not straight forward to use libreadline in this case (it
  might be possible to switch the state between different instances
  but that gets very cumbersome).

- readline does support asynchronous input via a special callback
  interface and can therefore be integrated with the
  scheduler. However, it does NOT support the same for output
  .. output operations might block.

- libedit/libeditline does not support scheduler integration

- SLang (SLrline) assumes, it is connected to a local terminal

If you know of another library, please check it out and check, wether
it can support

- callbacks for input *AND* output (scheduler integration)

- does not assume direct tty access

- allows to run multiple instances
