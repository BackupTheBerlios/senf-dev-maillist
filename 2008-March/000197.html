Summary: SystemException details should be streamlined

Details: Current usage of SystemExceptions (inside and outside of senf) shows, that the intended string argument ("where") is not always used as intended. Furthermore, the output of 'SystemException("bla") << "more bla" gives garbled output.

Proposed Solution: remove the (informal) semantic of the constructor argument (simply don't call it where but e.g. info or detail), and print errno + strerror text at the very start of ouput.

Still, care has to be taken on the final look of the exception output (e.g. punctuation, case, ...) on construction of a SystemException and when '<<'-ing more text to it.

Follow-Ups:

Date: 2008-Mar-06 15:18
By: jkaeber

Comment:
SystemException no longer assigns semantic to the constructor string argument (formerly "where", now "descr"). The argument is simply used for initial filling of the string returned by the what() member.

The member function "description()" was renamed to "errorString()".

The format of the string returned by what was slightly changed, e.g.
"[Permission denied] Could not do this. Some feature failed." etc.
