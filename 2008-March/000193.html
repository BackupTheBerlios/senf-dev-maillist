On the topic of the system exception errors:

How about starting every 'extended' information with a '\n'
character. This will produce nice multi-line error messages. 

Another thing; The current code *does* output errno and strerror
value. I did not get that part of the bugreport.

And lastly, all said applies to ALL senf exceptions (they all derive
from senf::Exception witch provides the error-message-extension
capability), so whenever extending an error message, one should cath
'senf::Exception' and not 'senf::SystemException'
