To whom it may concern, :-)

the current directory structure in SENF is not optimal:

* source code folders (PPI/, Socket/, ...) are mixed with non-source code
  stuff (debian/, _temlates/, ...)

* we have an include/senf/ folder with symlinks all over the place


To fix these problems, I have started to move things around in my
local repository in the following way:

* Added a senf/ folder to the checkout root. Moved all source code
  into this folder (PPI/, Socket/, config.hh, ...)

* removed the include/ folder.

* changed the include path in SConstruct from '#/include' to '#'

* will change *all* relative includes throughout senf with absolute
  includes.


This nicely fixes the problems listed above. It would be an option to
add another level src/ and move the boost/ and senf/ folders there. I
deem this unnecessary.

Of course, all this is open for discussion.

stefan.
